========================
GIT FILE STATUS SYMBOLS:
========================
-> U (Untracked)   : A brand-new file you just created. Git sees it 
                     but isn't watching its history yet.
                   
-> A (Added/staged): A new file that you ran 'git add' on. It is 
                     staged and ready to be saved in your next commit.
                   
-> M (Modified)    : A file Git already tracks, but you have edited 
                     or changed the text inside it.
                   
-> D (Deleted)     : A tracked file that you deleted. Git notices 
                     it's missing and waits for your commit.

-> C (Conflict)
   (A file with a merge conflict; happens when local and remote changes clash)

===================
GIT BASIC COMMANDS:
===================
-> git clone <repository-link>
   (Copies an existing repository from GitHub down to your computer)

-> git status
   (Checks the current state of your folder; shows untracked or modified files)

-> git add <filename>
   (Stages a specific file, preparing it to be saved; changes status from U to A)

-> git add .
   (Stages ALL new, modified, and deleted files in the current folder at once)/
   (adds all files-shortcut)

-> git commit -m "your_message"
   (Saves your staged snapshot permanently into Git's history with a short message)

-> git push origin <branch-name>
   (Uploads all your saved local commits from your computer up to GitHub)

-> git pull origin <branch-name>
   (Downloads the latest changes from GitHub down to your local computer)

-> git init
   (Initializes a brand new, empty local Git repository in your current folder)

-> git remote add origin <URL>
   (Links your local Git repository to your remote repository on GitHub using the nickname "origin")

-> git pull --rebase origin <branch-name>
   (Downloads the latest changes from GitHub AND replays your local commits 
    on top of them, keeping a clean, linear project history without extra merge commits)

-> git push -u origin main
   (Uploads commits to GitHub AND sets "main" as your default tracking branch for future shortcuts)

========================
 GIT BRANCHING COMMANDS:
========================

-> git branch
   (Lists all local branches in your project; puts an asterisk * next to your current branch)

-> git branch <branch-name>
   (Creates a brand new branch with that name, but keeps you on your current branch)

-> git checkout <branch-name>
   (Switches your current workspace over to that specific branch)

-> git checkout -b <branch-name>
   (Shortcut: Creates a brand new branch AND switches you to it instantly)

-> git branch -m <new-name>
   (Renames your current active branch to a new name)

-> git branch -v
   (Lists your branches vertically showing the latest commit message and ID for each)

-> git branch -d <branch-name>
   (Deletes a branch safely; only works if you have already merged its changes 
    and deleting branch should not be current or active branch)

-> git merge <branch-name>
   (Combines the changes from that specific branch into your current active branch)

-> git diff
   (Shows the exact line-by-line differences between your current unsaved changes and your last commit)

=====================
 GIT RESET COMMANDS :
=====================
-> git reset <file-name>
   (Unstages a specific file, moving it out of the green preparation area without altering any text inside it)

-> git reset HEAD <file>
   (Unstages a specific file, taking it out of the green preparation area without touching your code text)

-> git reset HEAD~1
   (Undoes your very last commit, dropping those changes back into your staging area so you can re-commit)

-> git reset <commit-hash>
   (Default/Mixed: Rewinds history back to a specific past commit hash; unstages your newer edits but keeps your code text)

-> git reset --soft <commit-hash>
   (Rewinds history back to a specific commit hash but keeps ALL your subsequent code changes staged and ready)

-> git reset --hard <commit-hash>
   ( DESTRUCTIVE: Rewinds history back to a specific commit hash AND permanently deletes all code written after it)

-> git reset --hard
   ( DESTRUCTIVE: Forcefully wipes away all current unsaved edits, resetting files back to the exact state of your last commit)
